# Git ワークフロー

## コミットのタイミング
- ひとつの機能や修正がある程度まとまったら、細かくコミットする
- 1コミット = 1つの目的（例：「ヘッダーコンポーネント追加」「避難所一覧ページ実装」など）
- 作業途中の壊れた状態ではコミットしない

## ブランチ戦略
- `main` ブランチに直接コミットしない
- 作業は必ずフィーチャーブランチで行う：`git checkout -b feature/作業内容`

## プッシュ・PR・マージの手順
まとまった単位で作業が完了したら以下の手順を実行する：

```bash
# 1. 変更をステージング・コミット
git add .
git commit -m "変更内容を簡潔に説明"

# 2. フィーチャーブランチをプッシュ
git push origin feature/作業内容

# 3. PRを作成
gh pr create --title "PRのタイトル" --body "変更内容の説明"

# 4. PRをマージ
gh pr merge --squash --delete-branch
```

## コミットメッセージの書き方
- 日本語でOK
- 何をしたかを一行で書く
- 例：`避難所マップページを追加`、`警報APIとの接続を実装`、`ヘッダーのスタイルを修正`
